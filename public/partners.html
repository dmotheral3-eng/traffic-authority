How It Works

The translation path. Four stages, one edge box.

From sensor output to TMC-native protocol in under 100 milliseconds — entirely at the deployment site. No cloud hop, no middleware on the DOT network, no changes to the backend.

STAGE 01
Sensor Output
LiDAR emits modern data — JSON events, object lists, real-time telemetry — exactly as the manufacturer designed it. No firmware changes, no custom sensor config.
IN · JSON / TCP / UDP STREAM
STAGE 02
Edge Translator
Docker container on the site's edge processor decodes the stream and maps detections to the target protocol's data model. Config cryptographically signed and hardware-locked.
XLT MODULE · AIR-GAPPED
STAGE 03
Protocol Stream
Encoder emits a byte-perfect legacy stream — Z1 binary frames, SunGuide XML, NTCIP 1209 objects, or Wavetronix HD serial — verified against protocol spec to the byte.
OUT · TARGET PROTOCOL, NATIVE FRAMING
STAGE 04
TMC Accepts as Native
The ATMS polls and receives exactly what it expects from a legacy device. Zero backend changes, zero DOT-side software, zero network policy exceptions.
TMC-NATIVE · NO BACKEND CHANGES
